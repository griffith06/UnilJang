# IP 리메이크 비즈니스 모델 제안서

> **IP 보유사:** W사
> **개발/서비스사:** 당사
> **장르:** PC온라인 MMORPG 리메이크
> **당사 배경:** 게임개발 30년+, 상용 게임엔진 개발, 동접 17만 글로벌 서비스  성공 이력.

---

## 기본 제안 베이스 (공통)

### 개발비 구조

> **리메이크 전제:** W사로부터 원작 소스코드 + 아트 리소스 일체 인수인계

| 항목            | 신규개발 (참고)  | **당사 리메이크 Full AI** | 절감 (신규 대비) |
| ------------- | ---------- | ------------------- | ---------- |
| 아트 (원화/3D/UI) | 8~12억      | **2~3억**            | ~75%       |
| 클라이언트 개발      | 6~10억      | **3~4억**            | ~60%       |
| 서버 개발         | 4~6억       | **2~2.5억**          | ~55%       |
| QA            | 2~3억       | **0.5~1억**          | ~70%       |
| 기획/레벨디자인      | 2~3억       | **1~1.5억**          | ~50%       |
| 인프라 구축        | 1~2억       | **1억**              | ~40%       |
| **합계**        | **23~36억** | **10~12억**          | **~60%**   |

> 제안 개발비: **10~12억** — 팀 규모 10~12명 기준 번레이트 + 품질 마진 확보
> W사 절감분은 마케팅 재원으로 활용 가능

### 마일스톤 지급 일정 (3분할 / 12개월)

```
계약금        착수 시            30%
1차 마일스톤  알파빌드 (4개월)   35%
2차 마일스톤  베타빌드 (8개월)   20%
3차 마일스톤  출시 빌드 (12개월) 15%
```

> 계약금 30% 이유: 소스/리소스 인수인계 + 초기 팀 구성 현금 소요 집중

| 회차    | 시점    | 주요 납품물                           |
| ----- | ----- | -------------------------------- |
| 계약금   | D-day | 기술기획서, 개발환경 구축, 소스 인수인계 확인서      |
| 1차 알파 | 4개월   | 핵심 전투/이동/캐릭터 3종, 1개 존, 서버 기본 접속  |
| 2차 베타 | 8개월   | 전체 콘텐츠 80%, CBT 가능 빌드, AI QA 리포트 |
| 3차 출시 | 12개월  | OBT 빌드, QA 완료, 운영툴 인계, 서버 안정화    |

### Recoup 구조

```
W사 투자금(선급금) → 출시 후 매출에서 W사 선회수(Recoup)
→ Recoup 완료 후 RS(Revenue Share) 적용
※ Recoup 범위: 개발 선급금 + 인프라 운영비 한정 (마케팅비 제외)
```

### 퍼포먼스 인센티브 티어 (모델 공통)

| 조건           | 당사 RS 추가 |
| ------------ | -------- |
| 월 MAU 50만 달성 | +5%p     |
| 동접 5만 달성     | +5%p     |
| 글로벌 서비스 확장   | 별도 협의    |

## Case A. W사 주도형

> W사가 QA·인프라·서비스 전반 통제 / 당사는 개발에 집중

### 역할 분담

| 기능                     | 당사  | W사        |
| ---------------------- | --- | --------- |
| 클라이언트/서버 개발            | ✅   |           |
| Full AI 개발 (아트/코드)     | ✅   |           |
| Dev QA (기능/회귀)         | ✅   |           |
| **서비스 QA (성능/보안/LQA)** |     | ✅         |
| **클라우드 인프라 운영**        |     | ✅ (계정 소유) |
| **배포 파이프라인**           |     | ✅         |
| **라이브 패치 최종 결정권**      | 제안  | ✅         |
| 마케팅 / UA               |     | ✅         |
| 현지화                    |     | ✅         |
| CS / 운영                |     | ✅         |
| 인프라 비용 부담              |     | ✅         |

### 개발비 및 RS

**개발비: 10~12억 (3회 마일스톤)**

| 구간           | 당사      | W사      | 비고              |
| ------------ | ------- | ------- | --------------- |
| Recoup 전     | 15%     | 85%     | W사 10억 회수 중     |
| Recoup 후     | **40%** | **60%** | W사 인프라/운영 부담 반영 |
| 퍼포먼스 티어 적용 시 | 최대 50%  | 최소 50%  |                 |

> Recoup 완료 예상: 월매출 1.5억 기준 약 7~8개월

### 당사 리스크

- 인프라 가시성 없어 장애 대응 지연 가능
- W사 QA 반려 시 출시 일정 종속
- 라이브 운영 방향에 대한 통제권 제한

## Case B. 당사 기술PM 주도형

> 당사 기술PM이 클라우드까지 장악 / W사는 비즈니스·마케팅 담당

### 역할 분담

| 기능                   | 당사        | W사      |
| -------------------- | --------- | ------- |
| 클라이언트/서버 개발          | ✅         |         |
| Full AI 개발           | ✅         |         |
| **클라우드 인프라 설계/운영**   | ✅ 기술PM 총괄 | 비용 부담만  |
| **배포 파이프라인 (CI/CD)** | ✅         |         |
| **서버 모니터링 / 장애대응**   | ✅ On-call |         |
| **QA 전체 주도**         | ✅         | 검수 참여   |
| 보안 / 취약점 대응          | ✅         | 협의      |
| 마케팅 / UA             |           | ✅       |
| 현지화                  |           | ✅       |
| CS (기술이슈 지원)         | ✅         | ✅ 운영 주도 |
| 인프라 비용 부담            | 운영 / 청구   | ✅ 비용 지불 |

### 기술PM 역할

```
당사 기술PM
├── 클라우드 아키텍처 설계 (리전/존/오토스케일링)
├── AI 인프라 파이프라인 관리
├── 배포 승인 및 롤백 결정
├── W사에 인프라 비용 월 리포팅
└── W사 기술담당자와 주간 싱크
```

### 개발비 및 RS

**개발비: 10~12억 (3회 마일스톤) + 출시 후 운영비 월 0.3억 (W사 부담, Recoup 포함)**

| 구간           | 당사      | W사      | 비고          |
| ------------ | ------- | ------- | ----------- |
| Recoup 전     | 15%     | 85%     | W사 회수 중     |
| Recoup 후     | **50%** | **50%** | 당사 운영 책임 반영 |
| 퍼포먼스 티어 적용 시 | 최대 60%  | 최소 40%  |             |

> Recoup 완료 예상: 월매출 1.5억 기준 약 8~10개월 (운영비 누적 포함)

### 당사 강점

- 서버 가시성 확보 → 장애 즉시 대응 (동접 17만 운영 경험 직접 적용)
- AI 파이프라인 기술 내재화 유지
- 라이브 서비스 품질 통제권 확보

## Case C. 당사 풀서비스형

> 당사가 Full AI로 개발·운영 전담 / W사 = IP 제공 + 현지화 + 마케팅만

### 역할 분담

| 기능                           | 당사     | W사   |
| ---------------------------- | ------ | ---- |
| 클라이언트/서버 개발                  | ✅      |      |
| **Full AI 개발** (아트/코드/QA자동화) | ✅      |      |
| **클라우드 인프라 (자체 계정)**         | ✅      |      |
| **배포 / 운영 / 장애대응**           | ✅      |      |
| **QA 전체 (AI 자동화)**           | ✅      |      |
| **CS / 운영툴**                 | ✅      |      |
| **빌링 / 결제**                  | ✅      |      |
| 현지화 (LQA)                    |        | ✅    |
| 마케팅 / UA / PR                |        | ✅    |
| **IP 라이선스**                  | 사용료 지급 | ✅ 수취 |

### Full AI 활용 포인트

```
개발 단계
├── AI 아트 생성 (원화 → 3D 에셋 자동화)
├── AI 코드 어시스트 (자체 파인튜닝 포함)
└── AI QA (자동 회귀테스트, 이상 감지)

운영 단계
├── AI 모니터링 (이상 트래픽/어뷰징 자동 탐지)
├── AI CS 1차 응대 (챗봇)
└── AI 밸런스 분석 (플레이 데이터 기반 자동 제안)
```

### 개발비 및 RS

**개발비: 10~12억 (3회 마일스톤) — 인프라 구축 포함, 운영비 당사 부담**

| 구간           | 당사      | W사      | 비고              |
| ------------ | ------- | ------- | --------------- |
| Recoup 전     | 15%     | 85%     | W사 12억 회수 중     |
| Recoup 후     | **63%** | **37%** | 당사 인프라/운영 전담 반영 |
| 퍼포먼스 티어 적용 시 | 최대 73%  | 최소 27%  |                 |

> Recoup 완료 예상: 월매출 1.5억 기준 약 9~11개월

### W사 설득 포인트

- 개발·운영 리스크 전혀 없음
- 투자금 회수 후 IP 수익 지속 발생
- 마케팅 성과 연동 인센티브 구조 제안 가능
- AI 파이프라인으로 빠른 업데이트 사이클 → IP 가치 상승

## 엔진 선택

### 비주얼 / 기술

| 항목          | 기존 엔진 대폭 개량      | UE5                                                |
| ----------- | ---------------- | -------------------------------------------------- |
| 렌더링 퀄리티     | 모던 피처 추가로 충분한 수준 | Lumen/Nanite 최상급                                   |
| 원작 게임플레이 재현 | 완벽 (소스 기반)       | 사실상 신규 개발                                          |
| 엔진 내부 장악력   | TD 직접 통제         | 블랙박스 영역 존재                                         |
| 장애 대응 속도    | 즉시               | 엔진 레벨 병목 가능                                        |
| 로열티         | **없음**           | 매출 5% 차감 (13억 초과분) / 커스텀 일시불 라이선스 대략 10억 ± a (비공개) |

### 빌드 시스템

| 항목            | 기존 엔진 대폭 개량              | UE5                        |
| ------------- | ------------------------ | -------------------------- |
| 빌드 시간 (PC)    | 수십 분 내외                  | 수 시간 ~ 하루                  |
| 빌드 시간 (모바일)   | **iOS + Android 1시간 미만** | **Android + PC 풀빌드 하루 소멸** |
| 빌드 머신 비용      | 일반 개발 PC                 | 1,000만원대 고사양 Mac 필수        |
| 빌드 실패 리스크     | 낮음                       | 셰이더 컴파일 등 변수 많음            |
| CI/CD 연동      | 자유도 높음                   | 커스텀 파이프라인 구축 필요            |
| 라이브 패치 속도     | 빠름                       | 빌드 병목이 핫픽스도 지연             |
| **개발 생산성 영향** | **영향 없음**                | **빌드 대기로 일과 단절 발생**        |

> 12개월 타임라인에서 빌드 시간 손실은 치명적. 하루 1회 빌드도 못 돌리면 이터레이션 속도 반토막.

### 개발 일정 리스크

| 항목          | 기존 엔진 대폭 개량 | UE5                |
| ----------- | ----------- | ------------------ |
| 12개월 완성 가능성 | **높음**      | 빡빡 (리스크 상존)        |
| 초기 적응 기간    | 없음          | 1~2개월 손실           |
| MMO 네트워킹    | 원작 그대로 재활용  | 커스텀 구현 필수          |
| 개발비         | **10~12억**  | **13~17억** (+3~5억) |

### 권장 결론

```
★ 기존 엔진 대폭 개량 (권장)
├── 10~12억 / 12개월 / 로열티 없음
├── 빌드 시스템 완전 장악 → 이터레이션 속도 유지
├── TD가 엔진 피처 직접 추가 → 기술 리스크 제로
└── 원작 서버/클라 로직 재활용 → 안정성 검증된 베이스

UE5 (참고안, W사 요청 시)
├── +3~5억 추가 = 13~17억
├── 비주얼 마케팅 임팩트 유리
└── 단, 빌드 시스템 / 일정 리스크 / 로열티가 12개월 딜에 불리
```
