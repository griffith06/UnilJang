##다이렉트X11 프레임웍 단계별 변경순서.
1. 기존 윈도우API프레임  -> 다이렉트 X11 기본 해상도 1280 x 960 창 모드로 세팅.창 잠김/오픈,풀스크린전환,alt+tab전환시 문제없게.
2. 800 x 600 텍스쳐 생성. 기존 스프라이트를 텍스쳐에 Copy 방식으로 처리.이후 프레임버퍼로 렌더링. Flush
3. 배경과 캐릭터를 텍스쳐로 얹는다.
배경 타일링된 부분을 1장 짜리 텍스쳐 uv_unwrap 하면 쉽지만 DP call수를 줄이기 위해 텍스쳐 아틀라스를 써야함.
또한 배경이 사각형이 아닌 쿼터뷰라 마름모 형태라서 알파로 찍게되면 알파 필터링문제로 타일을 어떤것을 먼저 찍느냐에 따라 픽셀이 이상해질수있음.
이부분은 아에 필터링없이 1:1로 RTT로 800x600에 찍고 그걸 strecching해서 고해상도에서 부드럽게 보이게 할까?
타일링 이슈 해결방법으로 끝도트들 2도트씩은 더 찍고 uv를 설정하면 될 듯. 
캐릭터는 헤어, 얼굴, 상,하체?(부정확함)  조합으로 찍음. sprite애니가 되고 있고, 해당부분도 DPcall수를 줄이기 위해 텍스쳐 아틀라스를 써야함.
기존 spr파일 해상도를 x2하면 1600x1200에 해당되는 리소스로 나오고 그 부분을 늘리고 줄이고 할때 느낌이 어떻게 될런지..
4. Direct Input은 사용안함. 기존 윈도우 메시지 핸들러로 처리함.
