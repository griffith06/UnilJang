# Godius Remaster — 텍스처 분류 & 리소스 추정

---

## 1. 텍스처 분류 체계

### 캐시 텍스처 (동적, Dynamic)
- **생성 시점**: 런타임에 필요할 때 생성/갱신
- **포맷**: ARGB8888 (비압축), D3D11_USAGE_DYNAMIC
- **갱신 조건**: 액션/방향/프레임 변경 시에만 (애니메이션 딜레이 ~130ms)
- **공유**: 동일 엔티티 타입 + 동일 프레임 → 캐시 히트로 공유

| 대상 | 이유 |
|------|------|
| 플레이어 | 장비 조합이 무한 (Head×Body×Leg×Weapon×Shield × 코스튬) |
| 몬스터 | 275종, 128.6MB RLE → 정적 로드 불가능 |
| NPC | 맵당 소수지만 동적 관리가 효율적 |

### 정적 텍스처 (Static, Pre-loaded)
- **생성 시점**: 맵 로드 시 미리 GPU에 업로드
- **포맷**: BC7 압축 (4:1 압축비) 또는 BC3 (알파 포함)
- **수명**: 맵 전환 시 해제

| 대상 | 이유 |
|------|------|
| 필드 오브젝트 | 맵당 고정, 종류 제한적, 변하지 않음 |
| 이펙트 (마법/스킬) | 미리 정의된 시퀀스, 공유 가능 |
| 아이템 (바닥 드롭) | 작은 스프라이트, 종류 제한적 |
| 트랩 | 맵당 고정 |
| 유저 건물 | 종류 제한적, 변하지 않음 |
| UI 텍스처 | 게임 시작 시 로드, 상시 유지 |

### 배경 타일 텍스처 (특수)
- **방식**: GPU 순환버퍼 (D3D11_TEXTURE_ADDRESS_WRAP 샘플러)
- **포맷**: ARGB8888 (비압축)
- **크기**: 928×608 (스크롤 버퍼 크기 그대로)
- **갱신**: 스크롤 시 가장자리 타일만 부분 업데이트 (UpdateSubresource)

## 6. 추후 최적화 포인트

| 최적화 | 설명 | 우선순위 |
|--------|------|----------|
| **동적 정점 배칭** | Y-정렬 순서대로 쿼드 누적, 텍스처 변경 시에만 DrawCall 분할 | 높음 (1차 구현) |
| **메가 아틀라스** | 동일 몬스터 종류의 프레임들을 큰 아틀라스에 합쳐 배칭 효율 향상 | 중간 (2차 최적화) |
| **인스턴싱** | 동일 텍스처 오브젝트를 인스턴스 렌더링. 단, Y-정렬로 인해 제한적 사용 | 낮음 (부분 적용) |
| **캐시 텍스처 LRU** | 사용 빈도 낮은 캐시 텍스처 자동 해제 | 중간 |
| **LOD (선택)** | 화면 가장자리 오브젝트 저해상도 렌더링 | 낮음 |

### 인스턴싱 제약사항
- Y-정렬 painter's algorithm 때문에 순수 인스턴싱은 사용 불가
- 같은 텍스처가 연속으로 나오는 구간에서만 부분적 인스턴싱 가능
- **동적 정점 배칭이 주력 최적화 방식**
