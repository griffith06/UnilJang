# Godius Client 프로젝트 폴더 구조 변경 계획

> **기준**: Next작업명세서.md §1(x64 전환), §2(SPR2+라이트맵+렌더기반+에디터) 작업 완료 상태 반영

---

## 완료 작업으로 인한 레거시 정리 대상

### §1 x64 전환 완료 → 삭제 대상

| 파일/폴더 | 사유 |
|-----------|------|
| `GodiusClient/DSWave.cpp/h` | XAudio2(AudioEngine)로 교체 완료. vcxproj에서도 제거 |
| `GodiusClient/DmidiPlay.cpp/h` | MCI MIDI → AudioEngine OGG로 교체 완료. vcxproj에서도 제거 |
| `GodiusClient/Dll.cpp/h` | 커스텀 PE 로더. 미사용 확인 후 제거 (§1-6) |
| `GodiusClient/CharBind.dll` | DLL 의존성 제거 완료 (§1-5). InitCharBind() 직접 호출로 변경됨 |
| `GodiusClient/GameGuard/` 전체 | NO_GAMEGUARD 정의, 게임가드 완전 제거 (§1-4) |
| `GodiusClient/GETDXVER.CPP` | Getdxver.h만 사용, 별도 CPP 불필요 확인 후 제거 |
| `cl/MSSDK7/` | x64에서 MSSDK7 참조 제거 완료 (§1-2). Windows SDK만 사용 |
| `cl/NPGameLib/` | GameGuard 제거로 NProtect SDK 불필요 |
| `GodiusClient/GcX.dsp, .dsw, .plg, .opt` | VC6 레거시 프로젝트 파일 |
| `GodiusClient/GcXWin.ilk, GcX.map` | 이전 빌드 산출물 |
| `GodiusClient/Script1.aps` | RC 자동 생성 캐시 |
| `GodiusClient/GcX.vcproj` | VC2005~2008 레거시 (vcxproj 사용) |
| `GodiusClient/가디우스 홈페이지.url.url` | 개발 불필요 |
| `GodiusClient/창모드실행 Test.bat` | render.ini로 대체 |

## 현재 구조 (Before)

```
cl/                              ← 프로젝트 루트
├── GodiusClient/                ← 클라이언트 (소스 + 리소스 + 실행파일 + 레거시 혼재)
│   ├── *.cpp, *.h               ← 251개 소스 + 신규 추가분, 폴더구분 없이 루트에 전부
│   ├── DSWave.cpp/h             ← ★ 레거시 (AudioEngine으로 교체됨)
│   ├── DmidiPlay.cpp/h          ← ★ 레거시 (AudioEngine으로 교체됨)
│   ├── Dll.cpp/h                ← ★ 레거시 (미사용 커스텀 PE 로더)
│   ├── CharBind.dll             ← ★ 레거시 (DLL 의존성 제거됨)
│   ├── GameGuard/               ← ★ 레거시 (게임가드 제거됨)
│   ├── Desc/                    ← 게임 데이터 정의 (34개 .DATA)
│   ├── Data/                    ← SPR 리소스
│   ├── Gm/, Map/, Object/, Sound/
│   ├── Json/                    ← jsoncpp 소스
│   ├── MiniDump/                ← 미니덤프 소스 2개
│   ├── Char/, CharSpr/, Costume/, WeaponShape/, User/ ← 비어있음
│   ├── Char.dat, Char.Off       ← 캐릭터 대용량 데이터 (409MB, Phase 5에서 폐기 예정)
│   ├── *.dll, *.ini, *.ver, *.ico
│   ├── *.sln, *.vcxproj, *.dsp, *.dsw  ← 신구 프로젝트 혼재
│   └── ReleaseX/                ← 비어있음
│
├── RESTools/                    ← 리소스 에디터 (소스 + 리소스 + 빌드산출물 혼재)
│   ├── *.cpp, *.h (78개 소스 + Dib32, Sprite2 신규)
│   ├── CHAR/, Spr/, Sprite/, Object/, Gm/, NPC/
│   ├── Map/, MapPcx/, MobLoc/, PCX/, Image_list/
│   ├── Data/, RES/
│   ├── Release/                 ← 빌드 중간파일 93MB+
│   ├── packages/                ← NuGet
│   ├── EF.exe, TClient.exe
│   ├── MFC42D.DLL, MFCO42D.DLL ← 레거시 시스템 DLL
│   └── *.sln, *.vcxproj, *.dsp, *.DSW
│
├── SPRToDDS/                    ← SPR→DDS 변환 유틸
├── SPRToSPR2/                   ← SPR→SPR2 변환 유틸
├── StringDataTool/              ← 문자열 데이터 도구
│   └── StringDataTool/          ← 중복 하위폴더
├── MSSDK7/                      ← ★ 레거시 (x64에서 불필요)
├── NPGameLib/                   ← ★ 레거시 (게임가드 제거됨)
└── utf8_source.props
```

## 폴더별 상세 설명

### Bin/ - 서비스 배포

**목적**: 최종 사용자에게 배포하는 게임 실행 환경. 이 폴더를 그대로 복사하면 게임이 돌아가야 함.

**포함 대상**:
- GcX.exe (x64 빌드 출력)
- 런타임 DLL (dbghelp.dll, 향후 steam_api64.dll)
- 설정 파일 (GodiusKR.ini, render.ini)
- 게임 데이터 (Desc/*.DATA, Data/, Map/, Sound/, Object/, Gm/)
- Char.dat/Char.Off (Phase 5 SPR2 전환 완료 전까지 유지)
- 컴파일된 셰이더 (Shaders/*.cso 또는 *.hlsl)

**제외 대상**:
- 개발 도구 실행파일 (EF.exe, SPRToDDS.exe 등) → DevBin/으로
- 소스코드, 프로젝트 파일
- 빌드 중간파일
- 레거시: ~~CharBind.dll~~, ~~GameGuard/~~

**vcxproj GcX 출력 설정**:
```xml
<OutDir>$(SolutionDir)\..\Bin\</OutDir>
```

### DevBin/ - 개발 배포

**목적**: 개발팀 내부에서 사용하는 도구 실행파일. 서비스 배포에는 포함되지 않음.

**포함 대상**:
- EF.exe (RESTools 리소스 에디터)
- SPRToDDS.exe
- SPRToSPR2.exe
- StringDataTool.exe
- TClient.exe (테스트 클라이언트)
- 도구 실행에 필요한 런타임 DLL (필요시)

**Bin과의 구분**:
| | Bin/ (서비스 배포) | DevBin/ (개발 배포) |
|---|---|---|
| 대상 | 최종 사용자 | 개발팀 내부 |
| 내용 | 게임 클라이언트 + 런타임 데이터 | 개발 도구 실행파일 |
| 배포 | 스팀/패치 서버 | 내부 공유/SVN |
| 크기 | 최소화 (배포 최적화) | 상관없음 |

**vcxproj 도구 프로젝트 출력 설정**:
```xml
<!-- EF, SPRToDDS, SPRToSPR2, StringDataTool -->
<OutDir>$(SolutionDir)\..\DevBin\</OutDir>
```

## 독립 작업 — Char.dat 폐기 + 캐릭터 SPR2 전환 (구조 변경과 병행 가능)

> Next작업명세서.md §1-5에서 CharBind.dll 제거 완료. 아래는 Char.dat 완전 폐기까지의 잔여 작업.
> **§3~§5 및 폴더 구조 변경과 의존성 없이 독립 진행 가능**. 우선순위 높음.

### 현재 상태

```
Char.dat (391MB, 비압축 아카이브)
├── Char.Off (637KB) — 인덱스: sFileBind × 22,487개 (fileName[21] + offset + size)
├── 내용: 캐릭터 부위별 SPR 22,487개 패킹
│   파일명 패턴: B{sex}{armor}{color}{action}.SPR (몸통), H{...} (머리), L{...} (다리), W{...} (무기)
├── 로딩: Sprite.cpp InitCharBind() → CharBindFileRead() → seek → SPR 로드
└── CharBind.dll: ✅ 제거 완료 (§1-5). InitCharBind() 직접 호출로 변경됨
```

**현재 렌더링 흐름:**
```
Char.dat(391MB) → seek → SPR 로드 → CPU 팔레트 변환 → 매 프레임 부위별 DrawClipping × 4~5회
```

### 목표

**목표 렌더링 흐름:**
```
부위별 SPR2(~15-30MB) → GPU 텍스처 로드 → 장비 변경 시에만 부위 합성 → 캐시 텍스처 1회 DrawCall
```

| | 현재 | 목표 |
|---|---|---|
| 저장 형태 | Char.dat 1개 아카이브 (391MB) | 부위별 멀티프레임 SPR2 ~1,500개 (~15-30MB) |
| 파일 수 | 22,487 SPR (액션별 개별) | ~1,500 SPR2 (부위+조합별 액션 통합) |
| 텍스처 포맷 | 팔레트 → CPU 변환 | BC7 GPU 네이티브 |
| 렌더링 | 매 프레임 부위별 DrawClipping × 4~5 | 캐시 텍스처 1회 DrawCall |
| 캐시 | 없음 | 장비 조합 해시 → 캐시 텍스처 (변경 시만 재합성) |

### 작업 단계

**Step 1: SPR→SPR2 일괄 변환기** (CLI 도구 → `DevBin/`)
1. Char.dat에서 22,487개 SPR 추출 (Char.Off 인덱스 기반)
2. 동일 부위(B/H/L/W) + 동일 조합의 액션(8종)별 SPR을 1개 멀티프레임 SPR2로 통합
3. BC7 압축 적용
4. 출력: `Bin/Char/` 폴더에 ~1,500개 SPR2 파일 배치

**Step 2: 클라이언트 로딩 경로 변경**
1. `LoadCharData()` 경로 변경: Char.dat seek → `Bin/Char/*.spr2` 개별 로드
2. SPR2 로더(§2에서 구현 완료)로 GPU 텍스처 생성

**Step 3: GPU 합성 캐시 시스템**
1. 장비 조합 키(성별+방어구+무기+머리 등) → 해시 → 캐시 텍스처 룩업
2. 캐시 미스: 부위별 SPR2를 RT에 순서대로 합성 → 캐시 저장
3. 캐시 히트: 캐시 텍스처 1회 DrawCall
4. 장비 변경/외형 변경 시에만 캐시 무효화 + 재합성

**Step 4: 레거시 제거**
1. `Char.dat`, `Char.Off` 삭제
2. `InitCharBind()`, `EndCharBind()`, `CharBindFileRead()` 코드 삭제
3. `RESTools/MapDoc.cpp`: Char.dat 읽기 → 개별 SPR2 로드로 변경

### 구조 변경 반영

구조 변경 완료 후의 Char 관련 파일 배치:

```
Bin/
├── Char/                          ← (Step 1 완료 후) 부위별 SPR2 ~1,500개
│   ├── B_M_A000_*.spr2           ← 남자 방어구0 몸통
│   ├── H_M_*.spr2                ← 남자 머리
│   ├── L_M_*.spr2                ← 남자 다리
│   ├── W_*.spr2                  ← 무기
│   └── ...
├── Char.dat                       ← (Step 4에서 삭제)
└── Char.Off                       ← (Step 4에서 삭제)
```
