# Godius Remaster — 렌더링 개선 세부 작업 계획표

---

## 전제 조건

- WorkFlow.md Phase 5~6 병행 시점부터 시작
- DX11 파이프라인, 캐시/정적 텍스처 시스템, Y-정렬 배칭이 완료된 상태
- 엔진 = 게임 클라이언트 (client), 툴 = RESTools (EF.exe)

### 비고 범례

| 구분 | 의미 |
|------|------|
| **엔진** | 게임 클라이언트 코드 작업 (렌더러, 셰이더, 게임 로직) |
| **에디터** | GcX.exe `-editor` 모드 — ImGui 독립 창 기반 인게임 에디터 (렌더링 파라미터 편집) |
| **툴** | RESTools(EF.exe) — SPR/SPR2 제작, 타일맵 편집, FGP 배치 등 리소스 제작 전용 |
| **동시** | 엔진과 에디터/툴 양쪽 모두 수정 필요 (또는 독립 병행 가능) |

### 툴-게임 프리뷰 전략

렌더링 관련 편집(광원, 물, 환경, 포스트프로세싱)은 RESTools가 아닌 **GcX.exe 에디터 모드**에서 수행한다. RESTools는 리소스 제작(SPR/SPR2, 타일, FGP)에만 집중한다.

| 역할 | 담당 | 이유 |
|------|------|------|
| SPR/SPR2 제작, 타일맵 편집, FGP 배치 | **RESTools (EF.exe)** | 기존 MFC 에디터 유지, CDib32 신규 클래스로 32bit 지원 추가 |
| 광원 배치/편집, 물 파라미터, 환경 프리셋, 포스트프로세싱 | **GcX.exe -editor** | 게임 렌더러에서 직접 확인, ImGui 독립 창 UI |

## Step 0: 에디터 모드 인프라 (선행)

RESTools와 게임 렌더링의 불일치 문제를 해결하기 위한 인게임 에디터 시스템.

| 0-1-3 | 핫리로드 기능 | `'r'` 키 → `.lgt`, `.cfg`, `.map` 등 현재 맵 데이터 재로드 | **엔진** |

### 0-2. ImGui Multi-Viewport 에디터 UI

| # | 세부 작업 | 설명 | 비고 |
|---|-----------|------|------|
| 0-2-1 | ImGui 통합 | ImGui docking 브랜치 도입, DX11 백엔드 연결 | **엔진** |
| 0-2-2 | Multi-Viewport 활성화 | `ImGuiConfigFlags_ViewportsEnable` — 에디터 패널을 OS 독립 창으로 분리 (게임 뷰포트 가리지 않음) | **엔진** |

- 밤낮/앰비언트 슬라이더 (Step 2)
- 물 파라미터 편집 (Step 4)
- 포스트프로세싱 파라미터 (Step 5)
- 환경 프리셋 편집 (Step 9)

### 0-3. RESTools ↔ GcX.exe IPC 연동

| # | 세부 작업 | 설명 | 비고 |
|---|-----------|------|------|
| 0-3-1 | Named Pipe IPC 채널 | `\\.\pipe\GodiusEditor` — RESTools → GcX.exe 커맨드 전송 | **동시** |
| 0-3-2 | IPC 프로토콜 정의 | 텍스트 기반 커맨드: `OPEN_MAP <name>`, `SCROLL_TO <x,y>`, `RELOAD_ALL` 등 | **동시** |
| 0-3-3 | RESTools 맵 에디터 연동 | 맵 열기/스크롤 시 IPC로 GcX.exe에 동기화 전송 | **툴** |
| 0-3-4 | GcX.exe IPC 수신 루프 | 메인 루프에서 Named Pipe 비동기 폴링, 커맨드 디스패치 | **엔진** |

**검증**: RESTools에서 맵 열기 → GcX.exe에서 동일 맵 자동 로드. RESTools 스크롤 → GcX.exe 동기 이동. RESTools 저장 → GcX.exe 자동 리로드.

## Step 2: 라이트맵 시스템 (#1)

밤낮 + 동적 라이트 + 분위기 연출의 코어 시스템. 가장 임팩트가 크다.

### 2-1. 라이트맵 기본 구조

| # | 세부 작업 | 설명 | 비고 |
|---|-----------|------|------|
| 2-1-1 | LightMap 클래스 설계 | CPU 측 ARGB 버퍼 (`render.ini` [Resolution] OriginalWidth/OriginalHeight의 1/2 또는 1/4), 광원 리스트, 앰비언트 색상 관리 | **엔진** |
| 2-1-2 | 앰비언트 클리어 | 매 프레임 라이트맵을 앰비언트 컬러로 초기화 (낮/밤/던전별 다른 값) | **엔진** |

**검증**: 화면 전체가 앰비언트 색상으로 어둡게/밝게 변하는지 확인. 마우스 위치에 테스트 광원 배치하여 동적 라이트 확인.

### 2-2. SPR2 기반 광원 리소스 시스템

광원 타입을 코드에서 개별 구현하지 않고, **맵툴에서 SPR2 리소스를 선택하여 배치·설정**하는 데이터 주도(data-driven) 방식으로 구현한다. 라이팅용 SPR2는 **ARGB8888(비압축) 포맷**을 지원해야 한다 (광원 그라데이션의 정확한 색상/알파 표현 필요).

| # | 세부 작업 | 설명 | 비고 |
|---|-----------|------|------|
| 2-2-1 | 광원 구조체 정의 | `LightInstance { position, spr2Name, frameIndex, color, scale, intensity, lifetime, timeOfDay, flickerParams, eventType }` | **엔진** |
| 2-2-2 | 광원 매니저 | 광원 추가/제거/갱신, 풀 기반 메모리 관리 (최대 256개) | **엔진** |
| 2-2-3 | 광원 배치 모드 | SPR2 리소스 선택 → 맵에 배치, ImGui 속성 편집 패널에서 설정 | **에디터** |
| 2-2-4 | 가로등 (Time of Day) | SPR2 리소스 배치 + 등장/소멸 시간대 설정 (예: 18:00~06:00) | **동시** |
| 2-2-5 | 횃불 / 플리커링 | SPR2 리소스 배치 + 플리커 파라미터 (주기, 강도, 노이즈) 설정 | **동시** |
| 2-2-6 | 스케일링 일렁임 | 일정 시간 동안 스케일이 변동하는 애니메이션 파라미터 설정 | **동시** |
| 2-2-7 | 타격 라이트 (이벤트) | 프로그램 내부에서 SPR2 이름 지정 → 이벤트 발생 시 생성/소멸 API 호출 | **엔진** |
| 2-2-8 | 마법 라이트 (이벤트) | SPR2 이름 지정, 마법 발사~소멸 이벤트에 연동, 이동 추적 | **엔진** |
| 2-2-9 | 번개 플래시 | 앰비언트 전체를 순간 밝게 (1~2프레임) | **엔진** |
| 2-2-10 | 이벤트 라이트 API | `SpawnLight(spr2Name, pos, params)` / `DestroyLight(handle)` — 코드에서 이벤트적으로 광원을 생성/소멸시키는 인터페이스 | **엔진** |

**맵툴 설정 항목** (SPR2 리소스 선택 후 편집):
- SPR2 리소스 파일명 / 프레임 인덱스
- 색상 틴트 (ARGB)
- 스케일 / 강도(intensity)
- Time of Day 등장/소멸 시간 (가로등 등)
- 플리커 여부 + 파라미터 (주기, 강도, 노이즈)
- 스케일링 일렁임 파라미터 (주기, 최소/최대 스케일)

**프로그램 내부 이벤트 호출 방식**:
- 타격, 마법 등 런타임 이벤트는 SPR2 이름을 지정하여 `SpawnLight()` / `DestroyLight()` API로 접근
- 맵에 사전 배치되지 않는 동적 광원은 모두 이 API를 통해 생성/소멸

**검증**: 맵툴에서 SPR2 광원 리소스 배치 → 게임에서 Time of Day, 플리커, 스케일링 동작 확인. 이벤트 API로 타격/마법 라이트 생성·소멸 확인.

## Step 3: 전투/타격 비주얼 피드백 (#2)

게임 플레이 체감 변화가 가장 큰 부분. 대부분 구현이 간단하다.

### 3-1. 화면 셰이크 (Screen Shake)

| # | 세부 작업 | 설명 | 비고 |
|---|-----------|------|------|
