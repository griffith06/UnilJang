# Godius Remaster — Features & System Requirements

## 목표
**원본 가디우스가 돌아가는 PC라면 리마스터도 문제없이 동작하는 수준.**

---

## 리마스터 주요 목표

| 항목 | 원본 | 리마스터 |
|------|------|----------|
| 렌더링 API | GDI + DirectDraw 7 | DirectX 11 |
| 해상도 | 800×600 고정 | 1280×960 기본, FHD/QHD/4K 지원 |
| 색상 | 256색 팔레트 (8-bit) | TrueColor ARGB8888 (32-bit) |
| 화면 비율 | 4:3 고정 | 와이드스크린(16:9, 16:10) 대응 |
| GPU 활용 | 없음 (CPU 소프트웨어 렌더링) | GPU 렌더링 (텍스처 기반) |
| 플랫폼 | 독자 런처 | Steam 연동 (Steamworks SDK) |

---

## 시스템 요구 사양

### 최저 사양 (Minimum)

| 항목 | 사양 |
|------|------|
| OS | Windows 7 SP1 (64-bit) |
| CPU | Intel Core 2 Duo E6600 / AMD Athlon 64 X2 5000+ |
| RAM | 2 GB |
| GPU | DirectX 11 지원, VRAM 256 MB (Intel HD 4000 / GeForce GT 430 / Radeon HD 5450) |
| 저장 공간 | 2 GB |
| 해상도 | 1280×960 |

### 권장 사양 (Recommended)

| 항목 | 사양 |
|------|------|
| OS | Windows 10 (64-bit) |
| CPU | Intel Core i3-4130 / AMD FX-4300 이상 |
| RAM | 4 GB |
| GPU | VRAM 512 MB 이상 (GeForce GT 730 / Radeon HD 7750) |
| 저장 공간 | 2 GB |
| 해상도 | 1920×1080 (FHD) |

## CPU 부하 추정

| 항목 | 부하 | 비고 |
|------|------|------|
| SPR→ARGB 변환 | 낮음 | DrawSprite 시 실시간 RLE→ARGB 변환 (1차), 느리면 LoadSpr 시 PreProcess 캐싱 (2차) |
| 캐시 텍스처 갱신 | 낮음 | 애니메이션 딜레이 130ms, 변경 시에만 |
| Y-정렬 (painter's algorithm) | 낮음 | 오브젝트 수백 개 수준 |
| 순환버퍼 타일 갱신 | 낮음 | 스크롤 시 가장자리만 갱신 |
| 네트워크 + 게임 로직 | 기존과 동일 | |

→ 원본 대비 CPU 부하는 **감소** (GPU 오프로드), GPU 부하는 **미미** (2D 쿼드 수백 장 수준)
