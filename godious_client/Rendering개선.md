# Godius Remaster — 렌더링 개선 (256컬러 → True Color)

---

## 개요

256컬러 팔레트 기반 → ARGB8888 True Color + D3D11 셰이더 전환으로 인한 변화 정리.

### 상위호환 대체하는 것

| 기존 기법 | 원리 | 대체 방안 |
|-----------|------|-----------|
| **물 팔레트 스크롤** | 팔레트 인덱스 시프트로 물 흐름 표현 | UV 스크롤 셰이더 (더 풍부한 표현 가능) |
| **팔레트 페이드** | RGB값 점진 축소 | 풀스크린 알파 폴리곤 (이미 계획됨) |
| **팔레트 색조 변환** | 팔레트 테이블 교체 | 픽셀 셰이더 컬러 그레이딩 |


### 새로 얻는 것

- 픽셀 셰이더 / 버텍스 셰이더 자유 사용
- 렌더 타겟 체이닝 (후처리 파이프라인)
- 알파 블렌딩 / 가산 블렌딩 자유자재
- 텍스처 샘플링 (UV 조작, 멀티 텍스처)
- 1670만 색상 + 8bit 알파 채널

---

## 1. 라이트 텍스처 시스템 (Light Map)

밤낮, 동적 라이트, 분위기 연출의 **코어 시스템**.

### 구조

```
화면 크기: 800×600 (또는 실제 해상도)
라이트맵 텍스처: 400×300(1/2) 또는 200×150 (1/4)
포맷: DXGI_FORMAT_R8G8B8A8_UNORM, D3D11_USAGE_DYNAMIC
```

### 렌더링 흐름

```
1. 라이트맵 클리어 → 앰비언트 컬러로 채움
   - 낮: (200, 200, 200) 밝은 회색
   - 밤: (30, 30, 60) 어두운 남색
   - 던전: (15, 15, 25) 짙은 어둠

2. 광원마다 라이트맵에 원형 그라데이션 스탬프
   - 중심 밝기 → 가장자리 0으로 감쇠 (거리 제곱 반비례)
   - 광원 색상 × 강도

3. 라이트맵 GPU 업로드 (Map/Unmap)

4. 최종 합성 (픽셀 셰이더):
   finalColor = screenColor × lightMapColor
   (Multiply 블렌딩, 바이리니어 필터링으로 자연스러운 경계)
```

### 광원 타입

| 타입 | 색상 | 반경 | 수명 | 설명 |
|------|------|------|------|------|
| **가로등/횃불** | 따뜻한 노란 (255,200,100) | 큼 (64~128px) | 영구 | 정적 포인트 라이트, 맵 데이터에 배치 |
| **캠프파이어** | 주황 (255,160,60) | 중간 (48~96px) | 영구 | 밝기 플리커링 (sin + noise) |
| **타격 라이트** | 흰색/빨강 | 작음 (24~48px) | 3~5프레임 | 생성→확장→소멸, 타격 위치에 스폰 |
| **마법 라이트** | 속성 컬러 | 중간 | 마법 지속시간 | 마법 이펙트에 부착, 이동 |
| **힐 라이트** | 초록/흰 (180,255,180) | 중간 | 짧음 | 캐릭터 중심에서 발산 |
| **번개** | 순백 (255,255,255) | 전체 화면 | 1~2프레임 | 앰비언트 전체를 순간 밝게 |
| **Light Shaft** | 노랑/흰 | 세로로 긴 타원 | 영구 | 가로등 위에서 아래로 길쭉한 스탬프 |

### 밤낮 사이클

```
시간대별 앰비언트:
  00:00~04:00  깊은 밤    (20, 20, 45)
  04:00~06:00  새벽       (80, 60, 80) → 분홍빛 그라데이션
  06:00~08:00  아침       (180, 170, 150) → 따뜻한 톤
  08:00~16:00  낮         (210, 210, 210) → 자연광
  16:00~18:00  석양       (200, 140, 80) → 주황빛
  18:00~20:00  초저녁     (80, 70, 100) → 보라빛
  20:00~24:00  밤         (30, 30, 60) → 남색

앰비언트 전환: Lerp 보간으로 부드럽게 (시간당 선형 보간)
실내 맵: 밤낮 사이클 무시, 맵별 고정 앰비언트
```

### 구현 난이도 / 부하 / 임팩트

- **복잡도**: 중간
- **부하**: 매우 낮음 (1/4 해상도면 200×150 = 30,000 픽셀 연산)
- **임팩트**: 최상 — 밤낮+동적라이트+분위기 한 번에 해결

## 2. 전투/타격 비주얼 피드백

게임 플레이 체감 퀄리티를 가장 크게 올리는 부분. 대부분 구현이 간단함.

### 2-A. 화면 셰이크 (Screen Shake)

카메라 댐핑 시스템에 셰이크 오프셋 추가.
```cpp
// 타격 시 셰이크 시작
shakeIntensity = 8.0f;  // 초기 진폭 (픽셀)
shakeDuration = 0.15f;  // 지속 시간 (초)

// 매 프레임 갱신
float decay = shakeIntensity * (remainingTime / shakeDuration);
float shakeX = (rand01() * 2 - 1) * decay;
float shakeY = (rand01() * 2 - 1) * decay;
cameraX += shakeX;
cameraY += shakeY;
```
- 타격 강도에 따라 진폭 조절
- 크리티컬/보스 공격: 강한 셰이크
- 일반 타격: 약한 셰이크 또는 없음

### 2-B. 히트 플래시 (Hit Flash)

피격 스프라이트를 1~2프레임 순백으로 렌더.
```hlsl
// 픽셀 셰이더
float4 finalColor = lerp(spriteColor, float4(1,1,1, spriteColor.a), flashIntensity);
```
- flashIntensity: 1.0 → 0.0으로 빠르게 감쇠
- 단순하지만 타격감에 기여 큼

### 2-C. 히트 프리즈 (Hit Pause)

타격 순간 2~3프레임 게임 일시정지 → 타격감 극대화.
```cpp
if (hitFreeze > 0) {
    hitFreeze--;
    return; // 게임 로직/애니메이션 갱신 스킵, 렌더링은 계속
}
```
- Dead Cells, Hades 등 액션 게임의 핵심 기법
- 강한 공격일수록 프리즈 길게 (1~5프레임)

### 2-D. 트레일 이펙트 (무기 궤적)

무기 스윙 시 궤적에 반투명 잔상 스프라이트.
```
프레임 N:   무기 위치 P0 (알파 1.0)
프레임 N-1: 무기 위치 P1 (알파 0.6)
프레임 N-2: 무기 위치 P2 (알파 0.3)
프레임 N-3: 무기 위치 P3 (알파 0.1)
```
- 링 버퍼로 이전 3~5프레임 위치 저장
- 잔상 색상: 무기 속성 컬러 (불=빨강, 얼음=파랑)

### 2-E. 스쿼시 & 스트레치 (Squash & Stretch)

점프/착지/타격 시 스프라이트 스케일 변형.
```cpp
// 착지 순간
scaleX = 1.2f;  scaleY = 0.8f;  // 납작하게
// → Lerp로 (1.0, 1.0)으로 복귀

// 점프 순간
scaleX = 0.85f; scaleY = 1.15f; // 세로로 늘림

// 타격 시 (공격자)
scaleX = 1.1f;  scaleY = 0.95f; // 약간 찌그러짐
```
- 버텍스 셰이더에서 쿼드 스케일 조정
- 미세한 변형 (±10~20%)이지만 생동감 큼

### 2-F. 데미지 넘버 물리

숫자가 튀어오르고 중력 적용되어 떨어짐.
```cpp
struct DamageNumber {
    float x, y;        // 위치
    float velX, velY;   // 속도 (초기: 위쪽 + 약간 랜덤 좌우)
    float gravity;      // 중력 가속도
    float alpha;        // 알파 (서서히 감소)
    int value;          // 데미지 값
    int type;           // 일반/크리티컬/힐/미스
};
```
- 크리티컬: 크고 빨간 폰트 + 높이 튕김
- 힐: 초록 + 위로 부드럽게 상승
- 미스: 회색 + 작게

### 2-G. 잔상 (After Image)

대시/빠른 이동 시 이전 위치에 반투명 복사본.
```
현재 위치: 원본 렌더
1프레임 전 위치: 알파 0.5 + 파란 틴트
2프레임 전 위치: 알파 0.25 + 파란 틴트
3프레임 전 위치: 알파 0.1 + 파란 틴트
```
- 은신/대시/텔레포트 스킬에 활용

### 구현 난이도 / 부하 / 임팩트

- **복잡도**: 대부분 낮음
- **부하**: 거의 없음
- **임팩트**: 최상 — 게임 플레이 체감 변화 가장 큼

## 3. 물 렌더링 (팔레트 스크롤 대체)

팔레트 스크롤 포기 대신 **굴절(Refraction) 기반** 물 표현으로 대폭 상위호환.

### 설계 방침

- 기존 타일 수정 불필요 — 물밑 바닥 타일(모래, 자갈, 해초 등)은 그대로 유지
- 물은 **별도 오버레이 패스**로 처리 — 바닥 타일 위에 물 쿼드를 덮어 렌더
- 물 쿼드 셰이더가 **아래 바닥을 굴절시켜 비침** → 물밑이 일렁이는 효과
- 물 영역 지정은 **맵 속성(CMapAttr) 비트플래그**로 처리

### 렌더링 구조

#### 렌더 순서

```
기존:   바닥 → 오브젝트/캐릭터
변경:   바닥 → 물 → 오브젝트/캐릭터
```

#### 상세 파이프라인

```
1. 바닥 타일 렌더 → 백버퍼 (기존과 동일, 물밑 바닥 포함)
   동시에: 바닥 결과를 1/4 축소 복사 → RT_PrevScene (200×150)
   (이전 프레임 것을 사용 — 1프레임 딜레이, 왜곡 표현이라 인지 불가)

2. 물 패스 → 백버퍼에 렌더
   - 맵 속성에서 dWATER_ATTR인 타일 위치에 쿼드 배치
   - 물 셰이더: RT_PrevScene에서 UV 왜곡 샘플링 (굴절)
   - 물 텍스처 + 색조 + 코스틱 합성
   - 모든 물 쿼드는 같은 텍스처 → DrawCall 1회

3. 오브젝트/캐릭터 → 백버퍼에 렌더

4. 후처리 (블룸, LUT 등)

5. UI → 백버퍼

6. Present
```

#### 이전 프레임 버퍼 (RT_PrevScene) 활용

```
매 프레임 끝:
  현재 바닥 RT (800×600) → 1/4 축소 복사 → RT_PrevScene (200×150)

다음 프레임:
  물 셰이더가 RT_PrevScene에서 굴절 샘플링

메모리: 200×150 × 4byte = ~117KB (거의 공짜)
라이트맵(200×150)과 같은 해상도 → 관리 깔끔

장점:
  - RT 전환 없음 (현재 프레임 바닥을 읽을 필요 없음)
  - 1/4 해상도 바이리니어 필터링 → 굴절이 자연스럽게 부드러움
  - 1프레임 딜레이 → 물밑 왜곡이라 차이 인지 불가능
  - 물 외 다른 왜곡 효과(열기, 폭발)에서도 재활용 가능
```

### 물 셰이더

```hlsl
// 물 픽셀 셰이더
Texture2D prevSceneTex;   // RT_PrevScene (이전 프레임 바닥, 200×150)
Texture2D waterTex;       // 물 표면 텍스처 (타일링)
Texture2D causticTex;     // 코스틱 노이즈 텍스처

cbuffer WaterParams {
    float time;
    float refractStrength;  // 굴절 강도 (0.003~0.01)
    float waterOpacity;     // 물 불투명도 (0.2 얕은물 ~ 0.7 깊은물)
    float causticIntensity; // 코스틱 밝기
    float3 waterTint;      // 물 색조 (바다=파랑, 늪=초록, 용암=빨강)
    float2 scrollSpeed;    // 물 흐름 속도/방향
};

float4 PS_Water(VS_OUTPUT input) : SV_Target
{
    float2 screenUV = input.pos.xy / screenSize;

    // 1. 굴절 UV 오프셋 (사인파 기반)
    float2 refractOffset;
    refractOffset.x = sin(input.uv.y * 30.0 + time * 2.0) * refractStrength;
    refractOffset.y = cos(input.uv.x * 25.0 + time * 1.7) * refractStrength * 0.7;

    // 2. 이전 프레임 바닥을 굴절시켜 샘플링
    float4 underwater = prevSceneTex.Sample(linearSampler, screenUV + refractOffset);

    // 3. 물 표면 텍스처 (듀얼 UV 스크롤)
    float4 water1 = waterTex.Sample(wrapSampler, input.uv + time * scrollSpeed);
    float4 water2 = waterTex.Sample(wrapSampler, input.uv * 1.3 - time * scrollSpeed * 0.7);
    float4 waterSurface = lerp(water1, water2, 0.5);

    // 4. 코스틱 반짝임
    float caustic = causticTex.Sample(wrapSampler, input.uv * 2.0 + time * 0.03).r;

    // 5. 최종 합성
    float3 result = lerp(underwater.rgb, waterSurface.rgb * waterTint, waterOpacity);
    result += caustic * causticIntensity;

    return float4(result, 1.0);
}
```

### 물 메쉬 구성

복잡한 메쉬 생성 불필요 — **타일 단위 쿼드 배칭**:

```cpp
// 매 프레임: 기존 타일 순회 루프에서 물 쿼드 수집
for (화면 내 타일) {
    if (mapAttr[tileX][tileY] & dWATER_ATTR) {
        AddWaterQuad(screenX, screenY, 64, 32);  // 아이소메트릭 쿼드
    }
}
// 물 쿼드 전부 같은 텍스처 → DrawCall 1회 (동적 정점 배칭)
```

- 화면 내 물 타일 50개 = 쿼드 50개 × 정점 4개 = 정점 200개 → **1회 DrawCall**
- 기존 오브젝트 배칭 시스템 그대로 사용

### 맵 에디터 — 물 속성 편집

기존 `CMapAttr` 속성 브러시 시스템을 **그대로 활용**.

#### 속성 비트 할당

```cpp
// MapDoc.h — 미사용 비트 활용
#define dWATER_ATTR    32   // 기존 dATTR_32 → 물 속성으로 사용
```

기존 속성 정의:
```
dEMPTY_ATTR    = 0    (없음)
dDONT_ATTR     = 1    (이동불가)
dEVENT_ATTR    = 2    (이벤트)
dMONAREA_ATTR  = 4    (몬스터영역)
dSAFEJON_ATTR  = 8    (안전지대)
dOUT_ATTR      = 16   (외곽)
dWATER_ATTR    = 32   (물)         ← 기존 dATTR_32 이름 변경
dATTR_64       = 64   (미사용)
dATTR_128      = 128  (미사용)
```

#### 편집 워크플로우

```
1단계: 바닥 타일 그리기 (기존과 동일)
  → 물밑 바닥: 모래, 자갈, 해초, 산호 등 원하는 타일 배치

2단계: 물 속성 칠하기
  → 작업 모드: dATTR_JOB 전환
  → 속성 선택: [물] 버튼 (기존 [Attr32] 버튼 이름 변경)
  → 마우스 드래그로 영역 선택 → 물 속성 페인팅
  → 비트 OR 연산으로 기존 속성과 중첩 가능 (이동불가+물 등)
```

#### 에디터 시각화

```
속성 표시 모드에서:
  물 속성 타일 → 반투명 파란색 오버레이로 표시
  기존 GetAttrColorByID()에 물 전용 컬러 추가

수정 필요 파일:
  MapDoc.h:     dATTR_32 → dWATER_ATTR 이름 변경
  MapDoc.cpp:   OnAttr32() 핸들러 이름/메뉴 텍스트 변경
                GetAttrColorByID()에 물 전용 팔레트 컬러 추가 (파란색)
```

#### 파일 포맷 영향

```
.atr 파일: 변경 없음 — WORD 배열 비트플래그, 기존 32번 비트 그대로 사용
.map 파일: 변경 없음
.cfg 파일: 맵별 물 설정 추가 (아래 참조)
```

### 맵별 물 설정

맵 설정(.cfg)에 물 파라미터 추가 → 맵마다 다른 물 느낌:

```cpp
struct MapWaterConfig {
    float opacity;        // 물 투명도 (0.2 얕은물 ~ 0.7 깊은물)
    float refractStrength;// 굴절 강도 (0.003 잔잔 ~ 0.01 거친)
    float2 scrollSpeed;   // 물 흐름 속도/방향
    float causticIntensity;// 코스틱 반짝임 강도
    DWORD tintColor;      // 물 색조
};
```

| 맵 환경 | opacity | refract | scrollSpeed | tint | 설명 |
|---------|---------|---------|-------------|------|------|
| **맑은 강** | 0.2 | 0.005 | (0.02, 0.01) | 연파랑 | 투명하고 잔잔 |
| **깊은 바다** | 0.6 | 0.008 | (0.03, 0.015) | 짙은파랑 | 불투명하고 출렁임 |
| **늪** | 0.5 | 0.003 | (0.005, 0.003) | 탁한초록 | 느리고 탁함 |
| **용암** | 0.7 | 0.01 | (0.01, 0.005) | 빨강/주황 | 불투명, 강한 왜곡 |
| **얼음 위** | 0.15 | 0.001 | (0.0, 0.0) | 하늘색 | 거의 투명, 정지 |

### 추가 디테일 효과 (선택)

| 효과 | 방법 | 비용 |
|------|------|------|
| **물결 파문** | 캐릭터가 물 위 이동 시 파문 스프라이트 스폰 | 낮음 |
| **코스틱 반짝임** | 밝은 노이즈 텍스처 UV 스크롤, 가산 블렌딩 | 낮음 |
| **테두리 거품** | 물-땅 경계에 거품 애니메이션 스프라이트 | 낮음 |
| **깊이별 색조** | 물 중심=짙은색, 가장자리=연한색 (속성맵 거리 기반) | 낮음 |
| **반사** | 물 위 오브젝트를 Y반전 반투명 렌더 | 중간 |

### 단계적 구현

```
1단계: 단순 오버레이 (RT 전환 없음)
  → 반투명 물 텍스처 + UV 스크롤 + 색조
  → DrawCall 1회 추가만으로 구현
  → 이것만으로도 팔레트 스크롤보다 나음

2단계: 굴절 추가 (RT_PrevScene 활용)
  → 이전 프레임 1/4 버퍼에서 왜곡 샘플링
  → 물밑이 일렁이며 비침 → 퀄리티 대폭 상승
  → 포스트프로세싱 파이프라인과 자연스럽게 통합

3단계: 디테일 (코스틱, 파문, 거품 등)
  → 기본 물 시스템 완성 후 점진 추가
```

### 구현 난이도 / 부하 / 임팩트

- **복잡도**: 1단계 낮음, 2단계 중간
- **부하**: DrawCall 1회 + RT_PrevScene 축소 복사 1회 (매우 낮음)
- **메모리**: RT_PrevScene 200×150 = ~117KB
- **임팩트**: 높음 — 팔레트 스크롤의 완전한 상위호환

## 4. 포스트 프로세싱 체인

D3D11 렌더 타겟 체이닝으로 후처리 파이프라인 구성.

### 렌더링 순서

```
씬 렌더 → RT_Scene
    ↓
[블룸] → RT_Bloom (선택)
    ↓
[왜곡] → RT_Distort (선택)
    ↓
[컬러 그레이딩] → RT_Final
    ↓
[비네트 + 오버레이] → 백버퍼
    ↓
UI 렌더 (후처리 영향 없음)
    ↓
Present
```

### 개별 효과

#### 4-A. 블룸 (Bloom)
밝은 영역에서 빛이 번져나오는 효과.
```
1. 밝기 임계값 이상 픽셀 추출 → 1/4 해상도 RT
2. 가우시안 블러 (가로 1패스 + 세로 1패스)
3. 원본에 가산(Additive) 합성
```
- 마법 이펙트, 광원, UI 하이라이트에 빛번짐
- 블룸 강도를 맵별/시간대별 조절 가능

#### 4-B. 컬러 그레이딩 (Color LUT)
16×16×16 3D LUT 텍스처로 전체 화면 색감 변환.
```hlsl
float3 lutCoord = color.rgb * (lutSize - 1.0) / lutSize + 0.5 / lutSize;
float3 graded = lutTexture.Sample(sampler, lutCoord).rgb;
```
- 맵별 분위기: 던전=어둡고 푸른, 사막=따뜻한, 숲=초록 강조
- 시간대별 LUT 전환 (밤낮 사이클과 연동)
- 상태이상: 독=초록 틴트, 저주=보라 틴트 (LUT 블렌딩)

#### 4-C. 비네트 (Vignette)
화면 가장자리를 어둡게 → 시선 집중 + 분위기.
```hlsl
float vignette = 1.0 - smoothstep(0.4, 1.0, length(uv - 0.5) * 1.2);
color.rgb *= vignette;
```
- 던전/실내에서 강하게, 필드에서 약하게
- 저체력 시 붉은 비네트 (경고 연출)

#### 4-D. 크로매틱 어버레이션 (Chromatic Aberration)
RGB 채널을 약간씩 오프셋 → 타격/데미지 순간 연출.
```hlsl
float2 offset = (uv - 0.5) * aberrationStrength;
float r = tex.Sample(sampler, uv + offset).r;
float g = tex.Sample(sampler, uv).g;
float b = tex.Sample(sampler, uv - offset).b;
```
- 평상시: OFF
- 강한 타격/피격 시: 1~3프레임만 활성화

#### 4-E. 모션 블러 (방사형)
대시/텔레포트 시 중심에서 바깥으로 스트리킹.
```hlsl
float2 dir = uv - center;
float4 result = 0;
for (int i = 0; i < samples; i++) {
    result += tex.Sample(sampler, uv - dir * i * blurAmount);
}
result /= samples;
```

#### 4-F. 필름 그레인 / 노이즈
미세한 노이즈 오버레이 → 레트로 감성, 어두운 맵 분위기.
- 타임 시드 기반 랜덤 노이즈
- 강도: 매우 약하게 (0.02~0.05)
- 밤/던전에서만 활성화 (선택)

### 구현 난이도 / 부하 / 임팩트

- **복잡도**: 중간 (렌더 타겟 체인 관리)
- **부하**: 블룸만 중간 (블러 2패스), 나머지 매우 낮음
- **임팩트**: 높음 — 블룸+LUT+비네트만으로 분위기 확 바뀜

## 5. 배경 오브젝트 셰이더 애니메이션

정적 오브젝트 (나무, 풀, 깃발)에 셰이더로 생동감 부여.

### 나무/풀 흔들림

```hlsl
// 버텍스 셰이더
float sway = sin(time * windSpeed + worldPos.x * 0.1) * swayAmount;
// 위쪽 버텍스만 적용 (아래쪽은 고정, 뿌리 고정)
float heightFactor = saturate(1.0 - uv.y); // UV 상단일수록 많이 흔들림
output.pos.x += sway * heightFactor;
```

### 오브젝트별 설정

| 오브젝트 | 흔들림 | 속도 | 세기 |
|----------|--------|------|------|
| **큰 나무** | 느리고 크게 | windSpeed = 1.5 | swayAmount = 3~5px |
| **작은 나무/관목** | 중간 | windSpeed = 2.5 | swayAmount = 2~3px |
| **풀/꽃** | 빠르고 작게 | windSpeed = 4.0 | swayAmount = 1~2px |
| **깃발/현수막** | 사인파 + 노이즈 | windSpeed = 3.0 | swayAmount = 4~6px |
| **횃불 불꽃** | 스케일 변동 | 빠름 | scaleY ± 10% |

### 글로벌 바람 시스템

```
windDirection: float2 — 바람 방향 벡터
windSpeed: float — 바람 세기 (0=무풍, 1=약풍, 3=강풍)
모든 흔들림 오브젝트가 같은 바람 벡터 참조 → 동기화된 자연스러운 움직임
날씨 시스템과 연동 가능 (비 올 때 강풍)
```

### 구현 난이도 / 부하 / 임팩트

- **복잡도**: 낮음 (버텍스 셰이더 수정만)
- **부하**: 거의 없음
- **임팩트**: 높음 — 정적인 배경이 살아 숨쉬는 느낌

## 6. 화면 왜곡 효과 (Distortion)

풀스크린 또는 부분 영역 UV 왜곡. 다양한 연출의 기반 시스템.

### 구현 원리

현재 프레임을 렌더 타겟에 렌더 후, 왜곡 셰이더로 후처리.
```hlsl
// 픽셀 셰이더
float2 center = distortionCenter; // 왜곡 중심 (정규화 좌표)
float2 dir = uv - center;
float dist = length(dir);
float factor = 1.0 + intensity * exp(-dist * dist * sharpness);
float2 distortedUV = center + dir * factor;
float4 color = screenTexture.Sample(clampSampler, distortedUV);
```

### 활용 목록

| 효과 | 왜곡 형태 | 지속 | 트리거 |
|------|-----------|------|--------|
| **타격 임팩트** | 타격 지점 중심 방사형 팽창 | 2~4프레임 | 강한 타격/크리티컬 |
| **보스 등장** | 화면 전체 진동 + 방사형 | 20~30프레임 | 보스 스폰 이벤트 |
| **워프 전환** | 소용돌이 (UV 회전) | 전환 동안 | 맵 이동/포탈 |
| **열기/아지랑이** | 위쪽으로 사인파 UV 오프셋 | 지속 | 사막/용암 맵 |
| **수중** | 전체 화면 물결 (사인파 x,y) | 지속 | 수중 맵/영역 |
| **포탈/마법진** | 특정 영역만 원형 왜곡 | 오브젝트 수명 | 포탈 오브젝트 |
| **피격 순간** | 화면 전체 약한 수축 | 1~2프레임 | 플레이어 피격 |
| **폭발** | 강한 방사형 확장 + 감쇠 | 5~8프레임 | 대형 마법/폭발 |

### 구현 난이도 / 부하 / 임팩트

- **복잡도**: 중간 (렌더 타겟 1장 추가 + 후처리 패스)
- **부하**: 낮음 (후처리 1패스)
- **임팩트**: 매우 높음

## 7. 파티클 시스템

CPU 기반 간단한 파티클. 2D 게임에서 GPU 파티클은 불필요.

### 파티클 구조체

```cpp
struct Particle {
    float x, y;           // 위치
    float velX, velY;      // 속도
    float life, maxLife;   // 수명
    float size;            // 크기
    float alpha;           // 투명도
    float rotation;        // 회전 (선택)
    DWORD color;           // 색상
    int textureIdx;        // 텍스처 인덱스
};
```

### 파티클 타입

| 타입 | 최대 수 | 크기 | 동작 | 맵 연동 |
|------|---------|------|------|---------|
| **비** | 100~200 | 1×8px 라인 | 위→아래 빠르게, 바람 영향 | 날씨 설정 |
| **눈** | 80~150 | 2~4px | 위→아래 느리게, 좌우 흔들림 | 날씨 설정 |
| **안개** | 10~20 | 64~128px 반투명 | 느리게 좌우 이동 | 맵별 설정 |
| **낙엽** | 20~40 | 4~8px | 위→아래 + 회전 + 좌우 | 가을 맵/숲 |
| **꽃잎** | 20~40 | 3~6px | 바람 방향 + 회전 | 벚꽃 맵 |
| **먼지/모래** | 30~50 | 1~3px | 바닥에서 위로 + 바람 | 사막/동굴 |
| **반딧불** | 15~25 | 2~4px + 글로우 | 랜덤 부유 + 밝기 변동 | 야간 숲 |
| **불씨/연기** | 20~30 | 3~8px | 위로 상승 + 알파 감쇠 | 화산/횃불 주변 |
| **발자국 먼지** | 캐릭터당 3~5 | 4~8px | 발밑에서 확산 + 소멸 | 흙/모래 타일 |
| **물 파문** | 캐릭터당 1~2 | 8~16px 원형 | 확산 + 알파 감쇠 | 물 타일 위 이동 |
| **전투 파티클** | 이펙트당 10~20 | 2~6px | 방사형 확산 | 타격/마법 |

### 라이트맵 연동

반딧불, 불씨 등 발광 파티클은 라이트맵에도 미약한 광원으로 등록 → 주변을 약간 밝힘.

### 구현 난이도 / 부하 / 임팩트

- **복잡도**: 중간 (파티클 매니저 + 이미터 시스템)
- **부하**: 100~200개 파티클은 CPU 부하 무시 가능
- **임팩트**: 높음 — 환경이 살아있는 느낌

## 8. 분위기 & 환경 연출

라이트맵(#1) + 포스트프로세싱(#4) + 파티클(#7)의 조합으로 맵별 분위기 설정.

### 맵 환경 프리셋

| 환경 | 앰비언트 | LUT | 파티클 | 특수 효과 |
|------|----------|-----|--------|-----------|
| **숲 (낮)** | 밝은 녹색 조 | 채도 높은 녹색 | 낙엽, 먼지 | 나무 흔들림, 빛 줄기 |
| **숲 (밤)** | 어두운 남색 | 차가운 파란 | 반딧불 | 달빛 광원, 비네트 강화 |
| **사막** | 밝은 황색 | 따뜻한 주황 | 모래 먼지 | 열기 왜곡 (아지랑이) |
| **던전** | 매우 어두움 | 차갑고 탈색 | 먼지, 거미줄 | 강한 비네트, 횃불 라이트 |
| **화산/용암** | 어두운 빨강 | 대비 높은 빨강 | 불씨, 연기 | 열기 왜곡, 바닥 붉은 광 |
| **눈 산** | 밝은 파란 | 차가운 파란+높은 밝기 | 눈 | 바람 강함 |
| **해변** | 밝은 하늘색 | 따뜻한 청색 | 물보라 | 물 셰이더, 파도 소리 |
| **묘지** | 어두운 보라 | 탈색+보라 틴트 | 안개 | 강한 안개, 유령 파티클 |
| **성안** | 따뜻한 실내 | 따뜻한 톤 | 먼지 | 창문 빛줄기, 촛불 |

### 날씨 시스템

```cpp
struct WeatherState {
    float rainIntensity;      // 0.0 ~ 1.0
    float snowIntensity;      // 0.0 ~ 1.0
    float fogDensity;         // 0.0 ~ 1.0
    float windSpeed;          // 바람 세기
    float2 windDirection;     // 바람 방향
    float ambientMultiplier;  // 앰비언트 밝기 배율 (흐린날 0.7)
    float thunderTimer;       // 번개 타이머
};
```

### 번개 연출
```
1. 랜덤 간격 (15~45초)으로 번개 타이머 발동
2. 1프레임: 라이트맵 전체 화이트 플래시 (앰비언트 = 255,255,255)
3. 2프레임: 원래 밝기로 복귀
4. 3~4프레임 후: 두 번째 약한 플래시 (더블 번개)
5. 사운드: 플래시 후 0.5~2초 딜레이로 천둥 재생 (거리감)
```

### 구현 난이도 / 부하 / 임팩트

- **복잡도**: 중간 (시스템 통합)
- **부하**: 낮음 (기존 시스템 조합)
- **임팩트**: 높음 — 맵 분위기 차별화

## 9. 셰이더 기반 스프라이트 효과

트루컬러 + D3D11 셰이더의 핵심 활용.

### 9-A. 아웃라인 셰이더
선택된 대상/NPC에 1~2px 컬러 아웃라인.
```hlsl
// 알파 채널 기반 엣지 검출
float alpha = spriteTexture.Sample(sampler, uv).a;
float alphaU = spriteTexture.Sample(sampler, uv + float2(0, -texelSize.y)).a;
float alphaD = spriteTexture.Sample(sampler, uv + float2(0, texelSize.y)).a;
float alphaL = spriteTexture.Sample(sampler, uv + float2(-texelSize.x, 0)).a;
float alphaR = spriteTexture.Sample(sampler, uv + float2(texelSize.x, 0)).a;
float edge = saturate(alphaU + alphaD + alphaL + alphaR) * (1.0 - alpha);
// edge > 0 이면 아웃라인 색상 출력
```
- 타겟 대상: 노란색 아웃라인
- 파티원: 초록 아웃라인
- NPC: 흰색 아웃라인
- 적대 플레이어: 빨간 아웃라인

### 9-B. 실루엣 렌더
오브젝트(나무, 건물) 뒤에 가려진 캐릭터를 반투명 단색으로 표시.
- 기존 Y-정렬에서 뒤에 가려지는 오브젝트 판별
- 가려진 영역만 반투명 단색 (팀 컬러 or 흰색) 으로 2차 렌더

### 9-C. 디졸브 이펙트
노이즈 텍스처 기반 디졸브 → 등장/사망/텔레포트.
```hlsl
float noise = noiseTexture.Sample(sampler, uv).r;
float threshold = dissolveProgress; // 0.0 → 1.0
if (noise < threshold) discard;
// 경계선에 발광 컬러 (dissolve edge glow)
float edge = smoothstep(threshold - 0.05, threshold, noise);
color.rgb += edgeColor * (1.0 - edge);
```
- 몬스터 사망: 아래에서 위로 디졸브
- 텔레포트: 전체 동시 디졸브
- NPC 등장: 역재생 (1.0 → 0.0)

### 9-D. 상태이상 셰이더

| 상태 | 셰이더 효과 |
|------|-------------|
| **독** | 초록 틴트 + 약간의 채도 감소 |
| **화상** | 빨강/주황 틴트 + 밝기 변동 (불꽃 느낌) |
| **빙결** | 파란 틴트 + 채도 감소 + 밝기 증가 (얼음 반사) |
| **석화** | 그레이스케일 + 약간의 콘트라스트 증가 |
| **투명화/은신** | 반투명 + 스캔라인 패턴 + 약간의 왜곡 |
| **버프/강화** | 외곽 글로우 (버프 속성 컬러) |
| **저주** | 보라 틴트 + 어둡게 + 노이즈 |
| **마비** | 노란 틴트 + 주기적 밝기 펄스 |

```hlsl
// 독 효과 예시
float3 poisonTint = float3(0.3, 0.8, 0.2);
color.rgb = lerp(color.rgb, color.rgb * poisonTint, poisonIntensity);
```

### 9-E. 팀 컬러/염색

셰이더로 특정 색상 범위를 다른 색상으로 교체.
- 길드 컬러 (망토/갑옷 일부)
- 장비 등급 글로우 (일반=없음, 레어=파랑, 에픽=보라, 전설=주황)

### 9-F. 스피드라인

대시/돌진 시 이동 방향으로 짧은 라인 파티클.
```
이동 방향의 반대쪽으로 5~10개 라인 스트릭
길이: 이동 속도에 비례
알파: 빠르게 감쇠
```

### 구현 난이도 / 부하 / 임팩트

- **복잡도**: 개별적으로 낮음
- **부하**: 거의 없음
- **임팩트**: 높음 — 상태이상/스킬 시각화 풍부

## 10. 화면 전환 이펙트

기존 페이드인/아웃 외 다양한 전환 연출.

### 전환 타입

| 전환 | 셰이더 원리 | 어울리는 상황 |
|------|-------------|---------------|
| **원형 와이프** | 중심~거리 기반 clip | 던전 진입/탈출 (레트로 RPG 클래식) |
| **다이아몬드 와이프** | `abs(uv.x-0.5)+abs(uv.y-0.5)` 기반 | 아이소메트릭에 어울리는 마름모 |
| **픽셀레이트** | UV를 floor로 양자화, 블록 크기 증가 | 꿈/회상/시간이동 |
| **블러 전환** | 가우시안 블러 강도 증가→감소 | 기절/부활 |
| **소용돌이** | UV를 중심 기준 각도 회전 | 포탈/워프 |
| **노이즈 디졸브** | 노이즈 < threshold → 신씬 | 일반 맵 전환 |
| **슬라이드** | 구씬/신씬 UV 오프셋 | UI 전환 |
| **줌 인/아웃** | UV 스케일 변경 | 컷씬 진입/탈출 |

### 원형 와이프 예시
```hlsl
float dist = length(uv - float2(0.5, 0.5));
float radius = wipeProgress; // 0.0 → 0.7 (닫힘), 0.7 → 0.0 (열림)
if (dist > radius) discard; // 또는 검정 출력
```

### 구현 난이도 / 부하 / 임팩트

- **복잡도**: 낮음 (각각 셰이더 1개)
- **부하**: 거의 없음
- **임팩트**: 중간 — 전환 연출 다양화

## 11. 그림자와 깊이감

### 11-A. 드롭 섀도우
캐릭터/몬스터 발밑 타원형 반투명 검정 스프라이트.
- 기본: 고정 크기 타원
- 개선: 라이트맵 밝기에 따라 그림자 진하기/방향 변경
- 점프 시: 그림자 크기 축소 (높이에 반비례)

### 11-B. 패럴랙스 레이어 (Parallax Scrolling)
배경을 2~3레이어로 분리, 카메라 이동 시 다른 속도로 스크롤.
```
Layer 0 (최후방): 하늘/산 — 카메라 속도 × 0.1
Layer 1 (후방):   먼 지형 — 카메라 속도 × 0.3
Layer 2 (메인):   게임 맵 — 카메라 속도 × 1.0
Layer 3 (전경):   나뭇잎/기둥 — 카메라 속도 × 1.3 (선택)
```
- 아이소메트릭 특성상 강한 패럴랙스는 부자연스러울 수 있음 → 미세하게 적용
- 야외 맵에서 하늘+구름 레이어 추가가 가장 효과적

### 11-C. 높이 기반 안개 오버레이
특정 맵 영역 (던전 입구, 계곡, 늪)에 지면 안개 오버레이.
- 반투명 안개 텍스처를 느리게 UV 스크롤
- 라이트맵 합성 전에 안개 레이어 추가

### 11-D. 공기 원근법 (Atmospheric Perspective)
화면 위쪽(=먼 곳) 오브젝트일수록 약간 탈색 + 안개 색상 블렌딩.
```hlsl
float depth = saturate(screenY / screenHeight);  // 0=상단(먼곳), 1=하단(가까운곳)
float fogFactor = smoothstep(0.0, 0.5, 1.0 - depth) * fogIntensity;
color.rgb = lerp(color.rgb, fogColor, fogFactor);
```

### 구현 난이도 / 부하 / 임팩트

- **복잡도**: 낮~중
- **부하**: 낮음
- **임팩트**: 중~높 — 특히 패럴랙스와 드롭 섀도우

## 12. 텍스처 블렌딩 (타일 경계)

아이소메트릭 타일 경계가 딱딱하게 끊기는 문제 해소.

### 기법

#### 12-A. 알파 마스크 블렌딩
타일 경계에 알파 그라데이션 마스크를 적용 → 인접 타일과 자연스럽게 섞임.
- 마름모(다이아몬드) 형태 마스크: 타일 가장자리 8~16px 영역만 페더링
- 인접 타일의 텍스처를 페더링 영역에서 블렌딩

#### 12-B. 트랜지션 타일
풀→흙, 흙→물 같은 전환 타일을 오토타일 규칙으로 자동 선택.
- RPG Maker 방식: 47패턴 오토타일 (지형 종류당 47장)
- 간소화: 4방향 + 4대각선 = 8패턴 트랜지션

#### 12-C. 디더링 블렌딩
레트로 느낌 유지하면서 체커보드/노이즈 패턴으로 블렌딩.
- 픽셀 좌표 기반 디더 패턴: `(x + y) % 2 == 0` → 타일A, 아니면 타일B

### 구현 난이도 / 부하 / 임팩트

- **복잡도**: 중간 (오토타일 로직이 핵심)
- **부하**: 낮음
- **임팩트**: 중간 — 배경 퀄리티 체감 큼

## 13. UI/UX 폴리시

### 13-A. 스무스 HP/MP 바

displayHP += (actualHP - displayHP) * 0.15f;

### 13-B. 대상 하이라이트

### 13-C. 데미지 넘버 연출

- 회피/미스: 작고 회색 + 위로 떠오름
- 힐: 초록 + 위로 부드럽게

### 13-D. 캐릭터 머리 위 표시
- 채팅 말풍선: 머리 위 + 서서히 페이드아웃

- 길드/파티 마크: 작은 아이콘

### 구현 난이도 / 부하 / 임팩트

- **복잡도**: 낮음
- **부하**: 거의 없음
- **임팩트**: 중간

## 14. 고급 기법 (선택적)

구현 난이도 있지만 차별화 효과 큰 것들. 후순위.

### 14-A. 노멀맵 2D 라이팅
스프라이트에 노멀맵 추가 → 포인트 라이트가 입체감 있게 반응.
- SpriteIlluminator 같은 도구로 기존 스프라이트에서 노멀맵 자동 생성
- 라이트맵 시스템과 결합하여 로컬 라이팅
- Eastward, Graveyard Keeper 등이 사용하는 기법
- **복잡도**: 높음 (노멀맵 에셋 제작 + 셰이더)

### 14-B. 서브픽셀 보간
캐릭터 이동 시 소수점 좌표에서 바이리니어 필터링 → 부드러운 움직임.
- 포인트 필터링: 1px 단위로 뚝뚝 끊김
- 바이리니어: 서브픽셀 위치에서 부드럽게 슬라이딩
- 픽셀아트 선명함 vs 부드러운 움직임 트레이드오프
- 옵션으로 제공 (레트로 느낌 vs 모던 느낌)
- **복잡도**: 낮음 (샘플러 스테이트만 변경)

### 14-C. SDF 기반 효과
Signed Distance Field로 소프트 섀도우, 부드러운 텍스트 렌더링.
- 텍스트 렌더링에 SDF 폰트 사용 → 모든 스케일에서 선명
- **복잡도**: 중간

### 14-D. GPU 스프라이트 아웃라인 (점프 플러드)
Jump Flood Algorithm으로 정확한 가변 두께 아웃라인.
- #9-A의 단순 엣지 검출보다 두꺼운 아웃라인에 적합
- **복잡도**: 높음
